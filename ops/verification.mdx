title: "对账"
description: "商户余额与链上资金的对账单，验证数据一致性。"
---


对账模块用于比对商户系统余额与链上实际资金，确保数据一致性。系统会自动计算差异金额并标注一致性状态。

## 页面说明

### 对账表格

| 字段 | 说明 |
|------|------|
| 日期 | 对账数据生成日期时间 |
| 所属项目 | 项目名称 |
| 商户总余额 | 系统记录的商户余额 |
| 链上余额 | 从区块链实际查询到的余额 |
| 待上链金额 | 已记账但尚未上链的金额 |
| 差异金额 | 商户总余额 - 链上余额 - 待上链金额 |
| 一致性 | 一致（绿色）/ 不一致（红色） |

### 差异金额计算

```
差异金额 = 商户总余额 - 链上余额 - 待上链金额
```

- **差异为 0**：显示为黑色，一致性标记为 ✅ 一致
- **差异为正数**：显示为绿色，表示系统余额大于链上余额
- **差异为负数**：显示为红色，表示链上余额超过系统记录

### 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 项目 | 下拉框 | 按项目筛选（超管可选所有项目） |
| 日期 | 日期选择 | 选择特定日期查看对账数据 |

## 操作指南

<Steps>
  <Step title="选择筛选条件">
    选择要查看的项目和日期。默认显示所有项目的最新对账数据。
  </Step>
  <Step title="检查一致性">
    关注「一致性」列，「不一致」的记录需要重点关注和排查。
  </Step>
  <Step title="分析差异">
    如果出现差异，检查：
    - 是否有待上链的交易正在处理中
    - 是否存在链上交易但系统未记录的情况
    - 是否有调账操作未同步
  </Step>
</Steps>

## 注意事项

<Warning>
  差异金额不为零不一定代表资金出问题。常见原因包括：链上交易延迟、待上链交易正在处理等。如持续存在差异，请及时排查。
</Warning>

<Tip>
  建议**每日定时检查**对账数据，及时发现和处理异常。可结合「报表」模块导出账单进行详细对账。
</Tip>

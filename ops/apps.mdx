App 列表模块用于管理接入 NUSD 平台的项目（应用）。每个 App 对应一个独立的商户项目，拥有独立的钱包、地址和费率配置。

## 页面说明

### App 表格

| 字段 | 说明 |
|------|------|
| 名称 | App 的显示名称 |
| 描述 | App 的说明信息 |
| apiPublicKey | API 公钥，用于接口签名验证 |
| webhook url | 回调通知地址 |
| 状态 | 正常（绿色）/ 停用（红色） |
| 创建时间 | App 创建时间 |
| 更新时间 | 最后更新时间 |
| 操作 | 编辑按钮 |

## 操作指南

### 添加 App

<Steps>
  <Step title="点击「添加」按钮">
    页面右上角点击「添加」按钮。
  </Step>
  <Step title="填写 App 信息">
    - **名称**（必填）：App 显示名称
    - **描述**（必填）：App 用途说明
    - **apiPublicKey**（可选）：商户的 API 公钥
    - **webhook**（可选）：交易状态回调 URL
  </Step>
  <Step title="确认提交">
    点击确定完成创建。
  </Step>
</Steps>

### 编辑 App

<Steps>
  <Step title="点击「编辑」按钮">
    在操作列点击「编辑」按钮。
  </Step>
  <Step title="修改信息">
    可修改名称、描述、apiPublicKey、webhook 和状态。
  </Step>
  <Step title="保存">
    点击确定保存修改。
  </Step>
</Steps>

## 权限要求

| 功能 | 超管(1) | 其他角色 |
|------|:------:|:------:|
| 查看 App 列表 | ✅ | ❌ |
| 添加 App | ✅ | ❌ |
| 编辑 App | ✅ | ❌ |

<Note>
  App 管理属于平台级配置，仅超管可操作。
</Note>

## 注意事项

<Warning>
  停用 App 后，该项目的所有交易功能（收款、支付等）将不可用。请确认后再操作。
</Warning>

<Tip>
  webhook URL 用于接收交易状态变更通知。配置后，充值和提现的状态变化会实时推送到该地址。
</Tip>

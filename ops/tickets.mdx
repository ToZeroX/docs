工单管理模块用于处理误充值退款等异常情况。商户可提交工单，超管审核后完成退款。

## 页面说明

### 工单列表表格

| 字段 | 说明 |
|------|------|
| 工单号 | 工单唯一标识（点击可跳转详情页） |
| 项目 | 所属项目 |
| 类型 | 工单类型（如「误充值」） |
| 状态 | 工单当前状态 |
| 币种 | 涉及的代币种类 |
| 充值金额 | 原始误充值金额 |
| 退款金额 | 审核确定的退款金额（扣除手续费后） |
| 创建时间 | 工单创建时间 |
| 操作 | 查看 / 审核 / 确认退款 / 重新提交 |

### 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 工单号 | 输入框 | 通过工单 ID 查询 |
| 状态 | 下拉框 | 全部 / 待审核 / 审核通过 / 已驳回 / 已完成 |
| 起止时间 | 日期范围 | 选择查询时间范围 |
| 项目 | 下拉框 | 超管可按项目筛选 |

## 操作指南

### 创建工单（管理员角色）

<Steps>
  <Step title="点击「创建工单」">
    页面右上角点击「创建工单」按钮。
  </Step>
  <Step title="输入充值哈希">
    填入误充值的交易哈希，点击「查询」按钮。系统会自动从链上查询交易信息并填充。
  </Step>
  <Step title="确认交易信息">
    查看自动填充的链、币种、金额、发送方/接收方地址。如果交易不属于当前项目，系统会提示项目不匹配。
  </Step>
  <Step title="填写退款地址">
    输入退款目标地址。
  </Step>
  <Step title="提交工单">
    填写备注（可选），点击确定提交。
  </Step>
</Steps>

<Warning>
  创建工单前请确保选择了正确的项目。如果交易哈希对应的项目与当前项目不一致，系统会阻止提交。
</Warning>

### 审核工单（超管）

<Steps>
  <Step title="找到待审核工单">
    在列表中找到状态为「待审核」的工单。
  </Step>
  <Step title="点击「审核」按钮">
    查看工单详情，包括充值金额和币种。
  </Step>
  <Step title="填写手续费">
    输入退款手续费，系统会自动计算实际退款金额（充值金额 - 手续费）。
  </Step>
  <Step title="选择审核结果">
    - 点击「审核通过」：工单进入 approved 状态
    - 点击「驳回」：工单进入 rejected 状态，可填写驳回原因

    如同时填写了「退回哈希」，工单将直接完成。
  </Step>
</Steps>

### 确认退款（超管）

审核通过后，需要在链上完成退款操作：

<Steps>
  <Step title="找到 approved 状态的工单">
    在列表中找到状态为「审核通过」的工单。
  </Step>
  <Step title="点击「确认退款」按钮">
    输入退款交易的链上哈希。
  </Step>
  <Step title="提交">
    点击确定，工单状态变为「已完成」。
  </Step>
</Steps>

### 重新提交（管理员角色）

被驳回的工单，商户可以查看驳回原因并重新提交：

<Steps>
  <Step title="点击被驳回工单的「重新提交」按钮">
    或进入工单详情页，在底部找到重新提交表单。
  </Step>
  <Step title="修改退款地址和备注">
    根据驳回原因修改相关信息。
  </Step>
  <Step title="提交">
    重新提交后，工单恢复为「待审核」状态。
  </Step>
</Steps>

## 工单详情页

点击工单号可进入详情页面，包含：

- **基本信息**：工单号、类型、项目、状态、链、币种、金额、手续费、充值哈希、来源地址、接收地址、退款地址、时间
- **费用明细**：充值金额、手续费、退款金额
- **退款交易**：退款的链上哈希
- **操作记录**：时间线展示所有操作历史（创建、审核、驳回等）
- **重新提交表单**：仅驳回状态的工单显示

## 注意事项

<Warning>
  审核时请仔细核对充值哈希、金额和退款地址。退款一旦执行无法撤销。
</Warning>

<Tip>
  审核时如果同时填写了「退回哈希」，可以跳过单独的「确认退款」步骤，工单直接完成。
</Tip>

<Note>
  工单详情页的操作记录包含所有历史操作，操作者邮箱已做脱敏处理（显示前 2 位 + **** + 后 2 位）。
</Note>
